# 典型 PM 提需求示例 · 场景 C：营销活动页

> 用途：展示"PM 用 AI 做营销活动页 / 落地页 / H5"类项目的 PM 说法 → 智能体响应
> 区别于场景 A/B，**强调文案微调、视觉对齐、活动期字段、响应式、多页面态**

---

## 示例 1：启动新项目（活动期紧急）

**PM 说**：
> 双 11 马上到了，我需要 3 个活动落地页：主会场 + 品类分会场 + 拼团专页。每个页面都有倒计时、优惠券领取、商品推荐。视觉稿 Figma 已有，希望 3 天内能跑起来。

**触发关键词**："我需要" + 时间压力 + 多个页面

**期望行为**：智能体开场 + 主动建议（页面优先级 / 哪些功能先做 / 设计稿输入确认）。

---

## 示例 2：明确要求扫描（已有项目结构）

**PM 说**：
> 帮我扫一下 src 目录，看现有营销页的页面结构、组件库、图片资源组织。

**触发关键词**："帮我扫"

**期望行为**：scan-agent 输出包含"页面清单 / 复用组件 / 图片资源 / 倒计时等通用组件"。

---

## 示例 3：要求改造（文案微调 — 轻量授权）

**PM 说**：
> 把主会场的标题从"双 11 主会场"改成"双 11 全球狂欢节"。

**触发关键词**："把 X 改成 Y"

**期望行为**：build-agent 改文案。**典型轻量授权场景**（单文件单字段文案修改），简化为只做 ① 授权检查，合并日报。

---

## 示例 4：要求规划（活动期字段管理）

**PM 说**：
> 帮我规划一下这些活动页里活动期字段怎么管理——倒计时、优惠券、库存、活动文案都是动态的。

**触发关键词**："帮我规划"

**期望行为**：plan-agent 输出**活动期字段分类**（静态文案 / 活动配置 / 实时数据 / 库存）。可能触发 PM 决策（字段后端化 vs 前端配置文件）。

---

## 示例 5：要求验收（视觉 + 4 态）

**PM 说**：
> 帮我验收一下品类分会场的视觉是不是跟 Figma 一致，倒计时 / 领券按钮 / 商品列表 / 空状态都跑一下。

**触发关键词**："帮我验收" + 多个验收点

**期望行为**：verify-agent 启动，**强制要求 PM 提供 Figma 截图** + 列出 4 态验收项（loading/empty/error/success）。

---

## 示例 6：超出授权范围

**PM 说**：
> 把主会场的"立即购买"按钮加上 hover 效果 + 点击埋点 + 分享到微信功能。

**智能体判断**：发现 3 个不同范围（UI 微调 / 数据埋点 / 第三方集成）。

**期望行为**：暂停 + 主动打断（`alignment` 标签）：
> "[alignment] 你授权：hover 效果 + 埋点 + 微信分享。我准备：(1) hover 样式（轻量）(2) 埋点 SDK 接入（需 PM 决策是否引入 SDK）(3) 微信 JSSDK 接入（需后端配合）。是否拆分执行？[仅 hover/hover+埋点/全做]"

---

## 示例 7：上下文不明确

**PM 说**：
> 把那个颜色调一下。

**期望行为**：智能体主动打断（`question` 标签），问 PM 具体是哪个页面、哪个元素、哪种调法（深一点？浅一点？换成品牌色？）。

---

## 示例 8：违反授权（不触发改文件）

**PM 说**：
> （在对话中讨论"这个优惠券样式不太好看"——未说"请帮我改"）

**期望行为**：智能体可以给建议（"建议增加阴影 + 加大字号"），但**不擅自改文件**，必须等 PM 说"请帮我改"。

---

## 场景特有注意点

- **时间敏感**：活动页通常有上线时间压力，智能体应在不违反治理机制的前提下，提示哪些可以并行、哪些可以并行验收
- **视觉还原度极高**：活动页对设计稿的还原度要求高于后台系统，verify-agent 的视觉对照段必须严格执行
- **活动期字段变更频繁**：倒计时、优惠券、库存等字段会随活动阶段变化，建议在 plan 阶段就设计"活动配置中心"概念
- **响应式必须**：活动页移动端访问占比 80%+，任何视觉对照都应同时给移动端 Figma
- **多页面态**：活动页的"已抢光 / 已结束 / 未开始"等状态对用户决策关键，4 态全覆盖是硬指标
