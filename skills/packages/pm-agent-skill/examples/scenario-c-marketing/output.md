## 示例 4：活动期字段决策（plan-agent 提请 PM 决策）

**触发**：plan-agent 完成活动期字段分析，发现需要 PM 决策

**智能体回执**：
```
[decision] 需要你拍板：活动期字段管理方案

选项：
- A: 前端配置文件（src/config/activity.ts，活动期改源码）
- B: 前端配置文件 + 远程配置中心（后端可改，前端拉取）
- C: 纯后端接口（前端无配置，每次都查）

推荐：B
理由：双 11 期间可能临时调整优惠券 / 倒计时截止时间，源码改 + 发版不现实；远程配置中心可让运营自助调整。

影响范围：
- 选 A：实现最快（静态文件）但运营不能调
- 选 B：需新增 1 个 mock 配置接口（前端可拉取） + 配置后台（PM/运营可编辑）
- 选 C：每次加载都查接口，性能略差但最实时

请选择：[A/B/C]
```

## 示例 7：发现反模式（活动配置硬编码）

**触发**：build-agent 改代码时发现既有反模式

**智能体回执**：
```
[risk] ⚠️ 发现既有反模式
位置：src/views/promotion/MainHall.vue（行 12~35）
反模式：anti-patterns/005（活动配置硬编码）
表现：倒计时截止时间、优惠券金额、库存数字全部硬编码在视图文件
影响：
- 活动期无法调整（需改源码 + 发版）
- 双 11 期间运营不能实时调

建议：
1. 抽到 src/config/activity.ts 配置文件
2. 通过 API 拉取（远程配置中心）
3. 是当前任务范围就改，不是就开 issue 跟踪

是否要 build-agent 顺手重构？[Y/n/开 issue 不改]
```
