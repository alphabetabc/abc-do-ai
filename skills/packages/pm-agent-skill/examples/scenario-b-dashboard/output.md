## 示例 4：权限矩阵决策（plan-agent 提请 PM 决策）

**触发**：plan-agent 完成模块边界设计，发现需要 PM 决策权限模型

**智能体回执**：
```
[decision] 需要你拍板：权限模型选型

选项：
- A: 路由级权限（每个路由只对特定角色可见，Tab 渲染前过滤）
- B: 字段级权限（同一 Tab 内不同角色看到不同字段，运营看不到金额）
- C: 混合（路由级 + 字段级，最灵活但实现成本高）

推荐：C
理由：你的需求是"运营看不到财务数据 / 财务看不到运营指标"——纯路由级会让"管理驾驶舱"类共享 Tab 难以实现；纯字段级会让独立 Tab 多绕一层判断。

影响范围：
- 选 A：实现快、Tab 切换快，但管理类 Tab 需要做角色判断逻辑
- 选 B：数据接口返回字段随角色变，后端 / mock 都需配合
- 选 C：多一层权限中间件（hook），代码组织稍复杂但最灵活

请选择：[A/B/C]
```

## 示例 7：发现反模式（数据 UI 混用）

**触发**：build-agent 改代码时发现既有反模式

**智能体回执**：
```
[risk] ⚠️ 发现既有反模式
位置：src/views/dashboard/RevenueTab.vue（行 45~78）
反模式：anti-patterns/003（数据 vs UI 混用）
表现：图表实例 + 业务数据 + 权限判断 + UI 状态 全部塞在 setup() 里
影响：阅读难 / 维护难 / 复用难

建议：
1. 拆分为 useRevenueData()（数据）+ useRevenueChart()（图表）+ useRoleGuard()（权限）3 个 hook
2. 视图组件只负责组装
3. 是当前任务范围就改，不是就开 issue 跟踪

是否要 build-agent 顺手重构？[Y/n/开 issue 不改]
```
