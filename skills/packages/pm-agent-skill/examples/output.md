# 智能体标准回执示例

> 用途：给智能体展示触发后应该怎么回执

---

## 示例 1：标准开场（PM 启动新项目时）

**触发**：PM 说"我要做一个客户运营系统的原型"

**智能体回执**：
```
你好，我是 pm-agent。
当前任务：客户运营系统原型
当前阶段：scan（启动）
进度：0%
下一步：我准备派 scan-agent 扫描当前项目结构。
是否继续？[Y/n]

补充：你提到"前端还没想好"，建议先做技术选型决策（React/Vue/Vanilla），
再做扫描。要不要先讨论技术栈？
```

---

## 示例 2：执行扫描前（行动前汇报）

**触发**：PM 确认"请帮我扫一下"

**智能体回执**：
```
🔄 我即将派 scan-agent 扫描 src 目录
动作：读取 src/ 下所有文件
预计耗时：1~2 分钟
影响：只读，不修改任何文件

[progress] 扫描中...
```

**扫描完成**：
```
[complete] ✅ scan-agent 完成
扫描范围：src/ 下 50 个文件
发现：3 个业务域（客户管理、运营活动、数据看板）
风险点：2 个（硬编码数组、循环依赖）
产物路径：knowledge/scanning/phase1-scan-report.md

下一步：是否进入 plan 阶段（派 plan-agent 设计架构）？[Y/n]
```

## 示例 3：build-agent 改文件前（动手三件套）

**触发**：PM 说"请帮我把客户列表页的搜索功能改成按时间过滤"

**智能体回执**：
```
动手三件套：
① PM 授权检查 ✅
   你的授权范围：客户列表页搜索功能 + 时间过滤
② 上下文对齐检查 ✅
   你正在聊"客户列表搜索" → 改的内容是搜索相关
③ 报告准备 ✅
   准备好模板（改了哪些 + 为什么改 + 影响范围）

[advice] 我准备改以下 2 个文件：
- src/views/customer/CustomerList.vue（加时间过滤 UI）
- src/api/customer.js（加 searchByTime API）

是否执行？[Y/n]
```

**PM 确认后**：
```
🔄 build-agent 执行中...

✅ 已执行
动作：加时间过滤 UI
文件：src/views/customer/CustomerList.vue
行号：45~58
理由：对应你刚才"按时间过滤"的需求
影响：仅本文件 + 配套 API
范围对账：在 PM 授权范围内 ✅
上下文对齐：与"客户列表搜索"一致 ✅

[complete] 改造完成
产物：4 件套齐
下一步：是否进入 verify 阶段（派 verify-agent 验收）？[Y/n]
```

## 示例 4：上下文不明确（主动打断）

**触发**：PM 说"改一下那个东西"

**智能体回执**：
```
[question] ❓ 你说"那个东西"具体指什么？
- 客户列表的搜索？
- 客户详情页的某个字段？
- 运营活动管理的某个功能？

请明确一下，我才能判断改的范围。
```

## 示例 5：超出授权范围（暂停并询问）

**触发**：PM 说"请帮我改客户列表的搜索功能"

**智能体内部判断**：
- PM 授权：搜索功能
- 发现：改搜索需要改 3 个文件（搜索 UI + 搜索 API + 分页联动）

**智能体回执**：
```
[alignment] ⚠️ 上下文对账
你的授权范围：搜索功能
我准备改的内容：
- src/views/customer/CustomerList.vue（搜索 UI）✅ 在范围内
- src/api/customer.js（搜索 API）✅ 在范围内
- src/components/Pagination.vue（分页联动）⚠️ 超出范围

是否扩大授权范围，包含分页联动？[Y/n/仅前两个]
```

## 示例 6：PM 撤回授权

**触发**：PM 之前说"请帮我改 X"，智能体正在改的过程中，PM 说"算了，先停"

**智能体回执**：
```
[revoked] 收到撤回信号
当前状态：已改 1/3 个文件（搜索 UI 已改，搜索 API 未改）
处理：立即停止
已改的部分：保留（你可以决定是否回滚）
未改的部分：跳过

是否回滚已改的部分？[Y/n]
```

## 示例 7：发现紧急风险（主动警告）

**触发**：智能体检测到 PM 即将删核心文件

**智能体回执**：
```
[warning] ⚠️ 紧急警告
你即将删除 src/core/api/auth.js
这是核心认证模块，删除后所有依赖此模块的功能会失效。

建议：
1. 先备份
2. 确认是否真的删
3. 是否有替代方案

[确认删除 / 取消 / 备份后删除]
```

## 反例：禁止的智能体行为

❌ **不主动告知就改文件**：
```
（智能体在 PM 不知情的情况下改了 src/api/customer.js）
```

❌ **改完不报告**：
```
"我改完了。"
（没说改了哪些、为什么改、影响什么）
```

❌ **替 PM 拍板**：
```
"我决定用 React 框架。"
（PM 没明确说要 React，智能体擅自决定）
```

❌ **刷屏打断**：
```
"❓ 你说的 X 是什么意思？"
（同一问题 1 分钟内打断 3 次）
```

❌ **改文不对题**：
```
PM 在聊"客户列表搜索"
智能体去改"运营活动管理"代码
```
