# PM × AI 协作 · 通用方法论手册 · 导读

> **定位**：沉淀"产品经理与 AI 协作做原型"的方法论与骨架。跨项目、跨技术栈、跨场景通用。
> **形态**：原则 + 骨架一体化，每条原则 1~3 行，原则卡片可独立成立、可拼装使用。
> **不绑**：本手册不绑定任何具体技术栈、文件路径、模块名、阶段编号。

---

## 整体框架

```
01 角色与边界       → 谁提需求 / 谁执行 / 谁决策
02 阶段化拆解       → Phase / 批次 / 端点 三层分解
03 标准化产物       → 每任务 4 件套 + 数据/UI 分类
04 执行 SOP         → 扫描→规划→改造→验收 闭环
05 文档体系设计     → 多文档分工 + 一致性 + 决策回溯
06 Bug 防御         → 反模式 + 案例归档 + 防御前置
07 决策白名单       → 自主范围 / 升级范围
08 PM 授权机制      → 智能体什么时候可以动手改文件
09 上下文对齐       → 改的内容必须符合 PM 当前聊天
10 主动打断         → 智能体什么时候可以打断 PM 对话
```

## 使用建议

- **每条原则独立成立**——删掉任意一条不会让其他条失效。
- **遇到冲突时**——以本手册为权威，结合具体项目语境调整。
- **跨场景复用**——本手册不绑定任何具体技术栈、文件路径、模块名。
- **持续迭代**——遇到新场景/新反模式，追加新原则而非推翻旧原则。

## 与具体项目的关系

本手册是**通用方法论**（套 A），与具体项目里的 `pm-agent` 设计文档（套 B）配合使用：
- 套 A → 讲"该按什么原则做"
- 套 B → 讲"pm-agent 这个具体智能体怎么实现"

两套独立可读，组合使用效果最佳。
