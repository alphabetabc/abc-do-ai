# 09 · 上下文对齐机制

> 核心命题：智能体执行动作时，怎么保证改的内容"文对题"？—— 必须验证动作内容 vs PM 当前聊天上下文

---

## 原则

**1. 默认对齐 PM 当前聊天上下文。**
- 智能体执行任何动作前，**必须验证**"动作内容 vs PM 当前聊天上下文"
- 不一致时，**先确认 PM**："你刚才说 X，我现在要做 Y，确认吗？"
- 一致时，才执行
怎么做：智能体维护"当前上下文快照"，每次动作前对账。

**2. 上下文的来源。**
- **PM 最近对话**：最近 5~10 轮 PM 说的话
- **PM 当前讨论的功能/模块/文件**：智能体实时跟踪
- **PM 当前任务的 PRD/skill**：智能体读取相关文档
- **最近一次授权**：PM 授权时的具体范围
怎么做：智能体持续维护"上下文快照"（最近 N 轮 + 当前任务 + 最近授权）。

**3. 对齐验证的时机。**
| 时机 | 验证内容 |
|------|---------|
| 每次"动手"前 | 改的文件/内容 vs PM 当前讨论 |
| 每次"派智能体"前 | 派的智能体类型 vs PM 当前任务 |
| 每次"重大决策"前 | 决策影响范围 vs PM 当前任务范围 |
| 每次"改配置"前 | 改的配置 vs PM 当前讨论的配置项 |
怎么做：智能体维护"动作前必做对账"清单。

**4. 对齐验证的方法。**
- **关键词匹配**：智能体动作的关键词 vs PM 聊天的关键词
- **任务关联**：智能体动作关联的任务 vs PM 当前任务 ID
- **文件关联**：智能体要改的文件 vs PM 当前讨论的文件
- **授权关联**：智能体动作 vs PM 最近一次授权的范围
怎么做：3 个维度（关键词 / 任务 / 文件）+ 1 个授权对账，全部通过才执行。

**5. 对齐失败的应对。**
- 智能体发现不一致 → **主动确认** PM
- PM 确认"就是要这样" → 智能体执行（更新上下文快照）
- PM 纠正"我说的是 X 不是 Y" → 智能体调整动作后执行
- PM 取消"算了" → 智能体不执行
怎么做：标准"不一致确认"模板。

**6. 上下文快照的更新。**
- PM 每说一句话 → 更新快照
- PM 切换话题 → 重置快照（保留最近 3 轮）
- PM 撤回授权 → 快照保留但标记"已撤回"
- 智能体执行动作 → 快照加上"动作结果"作为新上下文
怎么做：智能体持续维护"滚动上下文窗口"。

## 对账示例

### 示例 1：完全一致
```
PM: "我刚改了 customerListView 的搜索功能，加上按时间过滤"
智能体: "我准备改 customerListView 的搜索逻辑（按时间过滤）"
验证: 关键词匹配（customerListView / 搜索 / 时间过滤）✅
任务匹配（当前任务：customerListView 搜索优化）✅
文件匹配（customerListView）✅
授权匹配（PM 上一句"我刚改了" → 隐式授权智能体继续优化）✅
→ 执行
```

### 示例 2：关键词不一致
```
PM: "我刚改了 customerListView 的搜索功能"
智能体: "我准备改 operationDashboard 的 KPI 卡片"
验证: 关键词不匹配 ❌
→ 暂停 + 确认：
"你刚才说改 customerListView 搜索，我现在准备改 operationDashboard 的 KPI 卡片，与你说的不一致。是否确实要改 operationDashboard？"
```

### 示例 3：超出授权范围
```
PM: "请帮我改 customerListView 的搜索过滤逻辑"
智能体: 准备改 customerListView 的搜索 + 分页 + 排序
验证: 授权范围只包含"搜索过滤逻辑" ❌
→ 暂停 + 确认：
"你授权改搜索过滤逻辑。我准备顺带改分页和排序。是否扩大范围？"
```

---

## 反模式

- ❌ **擅自扩大范围**——改 X 时顺便改 Y
- ❌ **不验证上下文**——文不对题
- ❌ **验证了但不报告**——PM 不知道有验证
- ❌ **把"上下文相关"误判为"上下文一致"**——典型误用
- ❌ **PM 切换话题后不重置快照**——旧上下文影响新判断
- ❌ **撤回授权后用旧快照执行**——不尊重 PM 决策
- ❌ **验证失败但继续执行**——破坏可控性

## 实现细节

### 上下文快照结构
```yaml
context_snapshot:
  recent_pm_messages: [最近 N 轮 PM 消息]
  current_task: { task_id, task_name, scope }
  recent_authorization: { scope, granted_at, status }
  current_files_in_discussion: [文件路径列表]
  current_keywords: [关键词集合]
  last_update: 时间戳
```

### 对账流程
```
1. 智能体准备动作 A
2. 读取 context_snapshot
3. 三维度对账（关键词 / 任务 / 文件）
4. 授权对账（A vs 最近授权范围）
5. 全部通过 → 执行 + 报告
6. 任意一项不通过 → 暂停 + 确认 PM
```

### 上下文窗口大小
- 默认保留最近 10 轮对话
- PM 切换话题 → 保留最近 3 轮
- 超出窗口的旧对话 → 进入 `knowledge/decisions/` 作为历史
怎么做：智能体维护"滚动窗口"。

## 与其他机制的关系

| 机制 | 关系 |
|------|------|
| **PM 授权机制（08）** | 上下文对齐是"授权前对账"，授权是"对齐后执行" |
| **主动打断（10）** | 对账失败时主动打断 PM 确认 |
| **切换包机制** | 上下文快照随切换包一起交接 |
| **知识库** | 超出窗口的旧上下文进入 `decisions/` 作为历史 |
