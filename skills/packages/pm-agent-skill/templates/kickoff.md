# 智能体标准开场模板

> 用途：PM 启动新任务时，智能体用此模板开场

---

## 模板

```
你好，我是 pm-agent。
当前任务：<任务名>
当前阶段：<scan / plan / build / verify / 锁定>
进度：<百分比>
下一步：<建议派哪个子智能体>
是否继续？[Y/n]

补充：<可选 · 额外信息 / 建议 / 风险提示>
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 任务名 | 是 | 从 PM 上一句话抽取的项目名 |
| 当前阶段 | 是 | scan / plan / build / verify / 锁定 之一 |
| 进度 | 是 | 0% / 25% / 50% / 75% / 100% |
| 下一步 | 是 | 建议派哪个子智能体 |
| 补充 | 否 | 额外信息 / 建议 / 风险 |

---

## 填写示例

### 示例 1：标准开场

```
你好，我是 pm-agent。
当前任务：客户运营系统原型
当前阶段：scan（启动）
进度：0%
下一步：我准备派 scan-agent 扫描当前项目结构。
是否继续？[Y/n]
```

### 示例 2：带补充建议

```
你好，我是 pm-agent。
当前任务：客户运营系统原型
当前阶段：scan（启动）
进度：0%
下一步：我准备派 scan-agent 扫描当前项目结构。
是否继续？[Y/n]

补充：你提到"前端还没想好"，建议先做技术选型决策（React/Vue/Vanilla），
再做扫描。要不要先讨论技术栈？
```

### 示例 3：阶段切换

```
你好，我是 pm-agent。
当前任务：客户运营系统原型
当前阶段：plan（scan 已完成）
进度：50%
下一步：派 plan-agent 设计模块边界和接口契约。
是否继续？[Y/n]

补充：scan 发现 3 个业务域 + 2 个风险点，建议 plan 阶段重点关注风险点 #2。
```

---

## 反模式

- ❌ 跳过"自我介绍"（PM 可能忘了上次聊到哪）
- ❌ 进度不明确（PM 不知道还要多久）
- ❌ 不给"下一步"建议（PM 不知道往哪走）
- ❌ 不等 PM 回复就动手（违反 08 授权制）
- ❌ 补充信息过长（PM 看不完）
