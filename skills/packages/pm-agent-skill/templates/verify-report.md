# 验收报告模板

> 用途：verify-agent 验收 build-agent 产物时使用（不擅自回退，给 PM 建议）

---

## 模板

```
# 验收报告 · <phase 名>

## checklist 核对
- [✅/❌] 数据层完整
- [✅/❌] 接口层完整
- [✅/❌] 视图层完整
- [✅/❌] 配置层完整
- [✅/❌] 命名四联动一致
- [✅/❌] 数据 vs UI 分类正确
- [✅/❌] 不可序列化字段处理正确
- [✅/❌] 反模式未触发
- [✅/❌/⚠️ 不适用] 页面态 4 项齐全（loading/empty/error/success）

## 视觉 / 交互对照（高保真原型必填）
- [✅/❌/⚠️ 不适用] 布局与设计稿一致（截图对比）
- [✅/❌/⚠️ 不适用] 颜色 / 间距 / 字号 符合设计 token
- [✅/❌/⚠️ 不适用] hover / active / focus 状态齐全
- [✅/❌/⚠️ 不适用] 响应式断点（如适用）
- [✅/❌/⚠️ 不适用] 字体 / 图标与设计稿一致

## 历史 bug 回归
- [✅/❌] bug 案例 1：<名称>
- [✅/❌] bug 案例 2：<名称>
- ...

## 反模式防御验证
- [✅/❌] 反模式 1：<名称>
- [✅/❌] 反模式 2：<名称>
- ...

## 发现的问题 + 建议（不擅自回退）
- [ ] 问题 1：<描述>
  - 严重度：高/中/低
  - 位置：<文件:行号>
  - 建议：<具体建议>
  - 是否要 build-agent 改：<需 PM 决定>
- [ ] 问题 2：<描述>
  - 严重度：高/中/低
  - 位置：<文件:行号>
  - 建议：<具体建议>
  - 是否要 build-agent 改：<需 PM 决定>

## 验收结论（PM 决定）
- 建议结论：通过 / 不通过 / 部分通过
- 备注：<哪些是必须改的，哪些是建议改的>
- 上下文对齐：<验收标准 vs PM 当前任务是否一致>

## 等 PM 决策
- [ ] 是否通过验收？[Y/n]
- [ ] 是否要 build-agent 修问题？[问题编号/Y/全部]
- [ ] 是否锁定基线？[Y/n]
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| checklist 核对 | 是 | 7~8 项硬性 checklist |
| 历史 bug 回归 | 是 | 跑历史 bug 案例，每个打 ✅/❌ |
| 反模式防御验证 | 是 | 跑反模式清单，每个打 ✅/❌ |
| 发现问题 + 建议 | 是 | **不擅自回退**，给 PM 建议 |
| 验收结论 | 是 | 智能体**给建议**，**PM 拍板** |
| 等 PM 决策 | 是 | 必须等 PM 决策（不替 PM 拍板）|

---

## 填写示例

### 示例 1：完全通过

```
# 验收报告 · 客户运营系统原型 Phase 1

## checklist 核对
- [✅] 数据层完整（3 份数据契约）
- [✅] 接口层完整（3 个 API 客户端 + 3 套模拟数据）
- [✅] 视图层完整（3 个页面文件）
- [✅] 配置层完整（路由 + 菜单 + 权限）
- [✅] 命名四联动一致
- [✅] 数据 vs UI 分类正确
- [✅] 不可序列化字段处理正确
- [✅] 反模式未触发

## 历史 bug 回归
- [✅] bug-cases/phase1-001（命名不一致）：未复发
- [✅] bug-cases/phase1-002（硬编码数组）：未复发

## 反模式防御验证
- [✅] anti-patterns/001（数据 vs UI 混用）：未触发
- [✅] anti-patterns/002（不可序列化字段塞 API）：未触发

## 发现的问题 + 建议（不擅自回退）
（无）

## 验收结论（PM 决定）
- 建议结论：通过
- 备注：所有 checklist 通过 + 历史 bug 未复发 + 反模式未触发
- 上下文对齐：与 PM 任务"客户运营原型"一致

## 等 PM 决策
- [ ] 是否通过验收？[Y/n]
- [ ] 是否锁定基线？[Y/n]
```

# 验收报告 · 客户运营系统原型 Phase 1

## checklist 核对

- [✅] 接口层完整
- [✅] 视图层完整

- [✅] 不可序列化字段处理正确
- [✅] 反模式未触发

## 历史 bug 回归
- [✅] bug-cases/phase1-001：未复发
- [✅] bug-cases/phase1-002：未复发

## 反模式防御验证
- [✅] 全部未触发

## 发现的问题 + 建议（不擅自回退）
- [ ] 问题 1：缺国际化配置

  - 位置：src/i18n/ 目录

- [ ] 问题 2：客户列表未做空状态

  - 是否要 build-agent 改：需 PM 决定

## 验收结论（PM 决定）

- 备注：问题 1（中）建议修，问题 2（低）可后续迭代

### 示例 3：严重问题

```
# 验收报告 · 客户运营系统原型 Phase 1

## checklist 核对
- [✅] 数据层完整
- [❌] 接口层完整（searchByTime API 调用错误）
- [❌] 视图层完整（搜索框不显示）
- [❌] 命名四联动一致（search_by_time vs searchByTime 不一致）
- [✅] 数据 vs UI 分类正确
- [✅] 不可序列化字段处理正确
- [❌] 反模式触发：anti-patterns/003（命名四联动不一致）

## 历史 bug 回归
- [❌] bug-cases/phase1-003（命名不一致）：复发 ⚠️

## 反模式防御验证
- [❌] anti-patterns/003（命名四联动）：触发 ⚠️

## 发现的问题 + 建议（不擅自回退）
- [ ] 问题 1：searchByTime API 调用错误
  - 严重度：高
  - 位置：src/views/customer/CustomerList.vue（行 67）
  - 建议：调用 import { searchByTime } from '@/api/customer' 后调用
  - 是否要 build-agent 改：建议修
- [ ] 问题 2：命名四联动不一致
  - 严重度：高
  - 位置：多处
  - 建议：统一为 searchByTime（kebab-case 转 camelCase）
  - 是否要 build-agent 改：建议修
- [ ] 问题 3：搜索框不显示
  - 严重度：高
  - 位置：CustomerList.vue
  - 建议：检查 v-if 条件
  - 是否要 build-agent 改：建议修

## 验收结论（PM 决定）
- 建议结论：不通过（核心搜索功能失效）
- 备注：3 个高严重度问题，建议 build-agent 修复后重新验收
- 上下文对齐：与 PM 任务"客户搜索时间过滤"严重偏离 ⚠️

## 等 PM 决策
- [ ] 是否通过验收？[Y/n]
- [ ] 是否要 build-agent 修问题？[1/2/3/全部/n]
```

---

## 反模式

- ❌ 验收发现严重问题但放行（破坏基线质量）
- ❌ 擅自回退 build-agent（违反 08 授权制）
- ❌ 替 PM 决定验收结论（违反 §2.6 决策独立）
- ❌ checklist 走过场（漏项不被发现）
- ❌ 历史 bug 回归不跑（同 bug 复发）
- ❌ 反模式防御验证走过场
- ❌ 发现问题不落档（下次又踩）
- ❌ 不等 PM 决策就锁定基线
