# 执行报告模板

> 用途：build-agent 改完文件后，必出此报告（动手三件套第 ③ 件）

---

## 模板

```
✅ 已执行

业务语言摘要：<用 PM 能理解的话描述这次改了什么功能 / 解决了什么问题>
例：客户列表现在支持按注册时间筛选，并显示该时间段内的新增数量

动作：<改了哪些功能>
文件：
- <文件 1 路径>（行 <N>~<M>）：<改了什么>
- <文件 2 路径>（行 <N>~<M>）：<改了什么>
- <文件 3 路径>：<新建/修改/删除>

理由：<与 PM 哪句话 / 哪个需求对应>
影响：
- 直接影响：<文件 1>、<文件 2>、<文件 3>
- 间接影响：<依赖这些文件的其他模块>
- 风险评估：<低/中/高>

范围对账：
- PM 授权范围：<原话>
- 实际改的范围：<本次>
- 范围匹配：✅ 在范围内 / ⚠️ 超出范围（已询问 PM）

上下文对齐：
- PM 当前讨论：<关键词/任务>
- 改的内容：<关键词/任务>
- 对齐结果：✅ 一致 / ⚠️ 不一致（已确认 PM）

防御自检：
- [ ] 命名四联动检查
- [ ] 数据 vs UI 分类检查
- [ ] 不可序列化字段检查
- [ ] 防御自检通过
- [ ] 反模式未触发

下一步：<建议派哪个子智能体 / 等待 PM 决策>
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 业务语言摘要 | **是（高保真原型）** | 一句话用 PM 能理解的话描述这次改了什么功能 / 解决了什么问题；build-agent 强制输出 |
| 动作 | 是 | 技术角度一句话说清改了什么 |
| 文件 | 是 | 列出所有改动的文件（带行号、改了什么）|
| 理由 | 是 | 引用 PM 授权的原话 + 对应的需求 |
| 影响 | 是 | 直接影响 + 间接影响 + 风险评估 |
| 范围对账 | 是 | 必做（动手三件套第 ① 件的回头检查）|
| 上下文对齐 | 是 | 必做（动手三件套第 ② 件的回头检查）|
| 防御自检 | 是 | 列出 4~5 项自查 checkbox |
| 下一步 | 是 | 建议下一步动作 |

## 填写示例

### 示例 1：单文件小改

```
✅ 已执行

动作：客户列表页加时间过滤 UI
文件：
- src/views/customer/CustomerList.vue（行 45~58）：加时间选择器 + 过滤逻辑

理由：对应你刚才说"按时间过滤"的需求
影响：
- 直接影响：CustomerList.vue
- 间接影响：src/api/customer.js（需配套加 searchByTime API）
- 风险评估：低

范围对账：
- PM 授权范围："请帮我把客户列表页的搜索功能改成按时间过滤"
- 实际改的范围：客户列表页时间过滤 UI
- 范围匹配：✅ 在范围内

上下文对齐：
- PM 当前讨论：客户列表搜索
- 改的内容：客户列表时间过滤 UI
- 对齐结果：✅ 一致

防御自检：
- [x] 命名四联动检查
- [x] 数据 vs UI 分类检查
- [x] 不可序列化字段检查
- [x] 防御自检通过
- [x] 反模式未触发

下一步：派 build-agent 改 src/api/customer.js 加 searchByTime API（需 PM 确认）
```

### 示例 2：多文件改造

```
✅ 已执行

动作：实现客户搜索时间过滤功能（前后端）
文件：
- src/views/customer/CustomerList.vue（行 45~80）：加时间选择器 + 联动搜索
- src/api/customer.js（行 12~35）：加 searchByTime API 客户端
- mock/customer.js（行 50~80）：加时间过滤模拟数据
- src/components/TimeRangePicker.vue（新建）：时间范围选择组件

理由：对应你刚才说"按时间过滤"的需求
影响：
- 直接影响：上述 4 个文件
- 间接影响：搜索结果列表渲染、报表导出
- 风险评估：中（跨文件改动，需注意命名一致性）

范围对账：
- PM 授权范围："请帮我做时间过滤"
- 实际改的范围：4 个文件（含新组件）
- 范围匹配：⚠️ 略超出（新建组件需 PM 确认）

上下文对齐：
- PM 当前讨论：客户搜索优化
- 改的内容：客户搜索 + 时间过滤 + 新组件
- 对齐结果：✅ 一致

防御自检：
- [x] 命名四联动检查（searchByTime / time-range-picker）
- [x] 数据 vs UI 分类（time 数据入 API、UI 状态留 Vue）
- [x] 不可序列化字段检查（无 icon 引用）
- [x] 防御自检通过
- [x] 反模式未触发

下一步：派 verify-agent 验收
```

- PM 当前讨论：客户搜索 + 分页
- 改的内容：客户搜索 + 分页

防御自检：

---

## 反模式

- ❌ 缺理由（PM 不知道为啥改）
- ❌ 缺影响范围（PM 不知道改了会怎样）
- ❌ 缺范围对账（破坏 08 授权制）
- ❌ 缺上下文对齐（破坏 09 对齐机制）
- ❌ 缺防御自检（违反 04 执行 SOP）
- ❌ 防御自检走过场（checkbox 全打勾但实际没查）
