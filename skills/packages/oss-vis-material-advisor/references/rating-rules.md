# 评级规则（Rating Rules）

物料评级的**可执行规则手册**。本文件定义了任务 B（单物料评级）的全部判定逻辑。

## 评级流程

```
加载画像 profiles/{name}.json
        │
        ▼
按 6 个维度打分（0-5）
        │
        ▼
计算加权总分 = Σ(维度分 × 权重)
        │
        ▼
映射等级（A/B/C/D）→ 映射标签（🟢/🟡/🔴/⚫）
        │
        ▼
写回画像的 rating 字段
```

## 6 个评分维度

### 1. 业务描述力（权重 30%）

**问题**：这个物料在业务术语下能讲清一个什么样的故事？

| 分值 | 判定 |
|---|---|
| 5 | 物料名/schema 字段全是业务术语（如"销售总额""同比环比"），一看就懂用在什么业务 |
| 4 | 大部分是业务术语，少量技术词 |
| 3 | 业务术语和技术术语混用 |
| 2 | 主要是技术术语，需要解释才知道业务含义 |
| 1 | 完全是技术命名 |

**评分线索**：
- `oss-material.json.title` / `description` 是否业务化
- `schema.ts` 字段名是否业务化（`colorField` vs `salesColor`）
- `index.tsx` 注释 / 变量名是否业务化

### 2. 场景覆盖度（权重 20%）

**问题**：该类物料能覆盖该业务的几种典型变体？

| 分值 | 判定 |
|---|---|
| 5 | 同一物料能覆盖该业务 5+ 典型变体（如 echarts-bar 覆盖：销售排行、地区对比、部门业绩、库存预警、Top10） |
| 4 | 覆盖 3-4 种典型变体 |
| 3 | 覆盖 2 种典型变体 |
| 2 | 仅 1 种典型用法 |
| 1 | 单一固定展示 |

**评分线索**：
- 物料在 `materials/README.md` 分类下的复杂度标注
- schema 中是否有多种模式/类型切换（如 `chartType: bar | line | pie`）
- 是否支持多系列、多维度数据

### 3. 配置完备度（权重 15%）

**问题**：schema 配置项数量 + 是否有合理默认值 + 是否支持常用联动？

| 分值 | 判定 |
|---|---|
| 5 | 30+ 配置项 + 90% 都有合理默认值 + 支持字段联动 + 支持数据联动 |
| 4 | 20+ 配置项 + 80% 默认值合理 |
| 3 | 10-20 配置项，默认值覆盖一般 |
| 2 | <10 配置项，或大量必填无默认 |
| 1 | 配置项极少或缺失 |

**评分线索**：
- `schema.ts` 中 `properties` 的数量
- 每个字段是否有 `default` 值
- 是否有 `x-reactions`（字段联动）
- 是否有 `x-component` 多样化配置（颜色选择器、滑块、下拉等）

### 4. 数据契约清晰度（权重 15%）

**问题**：dataModel.json 是否规范、字段是否贴合真实业务流？

| 分值 | 判定 |
|---|---|
| 5 | dataModel 字段全部业务命名 + 有 mock 示例 + 有匹配说明 |
| 4 | 字段业务化，有 mock，匹配说明简单 |
| 3 | 字段业务化，缺 mock 或匹配说明 |
| 2 | 字段半业务半技术 |
| 1 | 字段是技术占位（`field1` / `value`） |

**评分线索**：
- `dataModel.json` 字段命名
- 是否有 `dataSource` mock 数据
- 是否有字段说明（注释或 `description` 字段）

### 5. 文档完整度（权重 10%）

**问题**：5+1 文档 / `doc/readme.md` 是否齐全？

| 分值 | 判定 |
|---|---|
| 5 | 5+1 全套（README/schema/component-logic/data-model/common-tasks/gotchas）+ `doc/readme.md` |
| 4 | 5+1 全套 |
| 3 | 5+1 部分（≥3 个）|
| 2 | 仅 README |
| 1 | 无任何文档 |

**评分线索**：
- 对照 `oss-vis-material-development-assistant/materials/{name}/` 目录
- 对照 `src/packages/{name}/doc/readme.md`

### 6. 可组合性（权重 10%）

**问题**：与同分类其他物料的组合代价？

| 分值 | 判定 |
|---|---|
| 5 | 与同类 80%+ 物料可直接组合（数据格式一致、尺寸/主题默认匹配） |
| 4 | 与同类 60%+ 物料可直接组合 |
| 3 | 与同类 40%+ 物料可直接组合 |
| 2 | 与同类 20%+ 物料可直接组合 |
| 1 | 与同类大多数不兼容 |

**评分线索**：
- 物料是否位于 `oss-chart-plots/plots/*`（共享 schema 高可组合性）
- 物料 schema 是否暴露 `width/height/theme` 等通用配置
- dataModel 是否采用通用格式（`{ name, value }` vs 私有结构）

## 等级映射

```
总分 = 业务描述力×0.3 + 场景覆盖度×0.2 + 配置完备度×0.15
     + 数据契约×0.15 + 文档完整度×0.1 + 可组合性×0.1

4.5 ≤ A    🟢 独立优秀
3.5 ≤ B < 4.5  🟡 组合可用
2.5 ≤ C < 3.5  🔴 组合复杂
D  < 2.5   ⚫ 不建议
```

## 标签判定补充规则

**仅看等级不够，还要看实际能力**：

| 情况 | 等级 | 标签 | 解释 |
|---|---|---|---|
| 总分 A 且 业务描述力 ≥ 4 | A | 🟢 独立优秀 | 单物料能讲清一个业务故事 |
| 总分 A 但 业务描述力 < 4 | A | 🟡 组合可用 | 能力齐备但业务表达力弱，需配合标题/标签/数字 |
| 总分 B 且 配置完备度 ≥ 4 | B | 🟡 组合可用 | 配置够用，组合代价低 |
| 总分 B 但 配置完备度 < 3 | B | 🔴 组合复杂 | 需深度定制配置 |
| 总分 C | C | 🔴 组合复杂 | 多种能力都欠缺 |
| 总分 D | D | ⚫ 不建议 | 不推荐使用 |

## 评分模板

在画像的 `rating.breakdown` 字段下输出：

```json
{
  "score": 4.2,
  "grade": "A",
  "label": "🟢 独立优秀",
  "breakdown": {
    "business_description": 4.5,
    "scenario_coverage": 4,
    "config_completeness": 4,
    "data_contract_clarity": 4,
    "doc_completeness": 5,
    "composability": 3.5
  },
  "weighted_total": 4.2,
  "reasoning": "1. 业务描述力 4.5：echarts-bar schema 字段多为业务化（categoryField/valueField），但仍有 'seriesField' 这类技术词。2. 场景覆盖度 4：支持横向/纵向、单系列/多系列，能覆盖 4 种典型业务。3. ...",
  "scored_at": "2026-06-16"
}
```

`reasoning` 字段**必须写**，给产品经理看评分依据，每个维度 1-2 句话。
