# 组合方案评级规则（Composition Rules）

物料组合方案（任务 C）的**可执行规则手册**。本文件定义了如何基于画像库推荐业务场景方案并评级。

## 任务目标

**输入**：业务场景描述（自然语言，如"销售大盘""设备运维监控"）
**输出**：Top 1-3 推荐方案卡片，每个含物料清单 + 组合代价 + 评级

## 工作流程

```
输入业务场景
    │
    ▼
步骤 1：场景关键词提取
    │
    ▼
步骤 2：物料候选池匹配（从 profiles/*.json 检索）
    │
    ▼
步骤 3：组合方案生成（贪心 + 约束求解）
    │
    ▼
步骤 4：组合代价计算（兼容性矩阵）
    │
    ▼
步骤 5：方案评级 + 输出卡片
```

---

## 步骤 1：场景关键词提取

从自然语言描述中提取**业务关键词**和**能力关键词**：

| 关键词类型 | 例子 | 用途 |
|---|---|---|
| 业务实体 | 销售、设备、用户、订单 | 匹配业务场景 |
| 行为动词 | 排行、对比、趋势、占比 | 匹配视觉类型 |
| 修饰词 | 实时、Top10、年度、同比 | 匹配配置变体 |

**关键词表**（持续维护）：

```yaml
业务实体 → 物料分类：
  销售/营收/业绩 → 图表（bar/pie/line）+ 数字卡
  设备/机器/资产 → 表格 + 地图 + 3D 拓扑
  用户/客户/会员 → 列表 + 排行榜 + 数字卡
  订单/交易/流水 → 表格 + 折线图
  地区/省份/城市 → 地图 + 排行
  时间/日程/排期 → 时钟 + 轮播

行为动词 → 视觉类型：
  排行/Top → 列表/排行榜/横向条形
  对比/差距 → 双向条形/分组柱状
  趋势/走势 → 折线图/面积图
  占比/构成 → 饼图/环图/玫瑰图
  分布/地理 → 地图
  进度/完成率 → 进度条
  监控/状态 → 仪表盘/状态卡
  装饰/美化 → 边框/装饰图形
```

---

## 步骤 2：物料候选池匹配

### 2.1 画像字段检索

加载所有 `profiles/*.json`，对每个画像做关键词匹配打分：

```json
{
  "candidate_pool": [
    {
      "material": "echarts-bar",
      "match_score": 0.85,
      "matched_keywords": ["排行", "对比", "销售"],
      "matched_scenarios": ["销售业绩 Top10 排行", "地区营收对比"]
    }
  ]
}
```

**匹配打分公式**（粗略）：
- 业务场景匹配 1 项 = +0.3
- 关键词匹配 1 项 = +0.2
- 画像 `category` 与场景分类匹配 = +0.3
- `match_score` < 0.3 剔除

### 2.2 候选池精简

- 同一分类（如同为"图表/ECharts"）保留 match_score Top 2-3
- 装饰类（边框/decoration）默认不放候选池，按需添加
- 容器类（free-layout-*）默认进候选池

## 步骤 3：组合方案生成

### 3.1 必备物料判定

**业务场景标准物料组合**（基础模板）：

| 场景类型 | 必备物料 | 可选物料 |
|---|---|---|
| 数据大盘 | 1-2 个数字卡 + 1-2 个图表 + 标题 + 边框 | 列表、轮播 |
| 监控驾驶舱 | 数字卡 + 仪表盘 + 实时表格 + 状态指示 | 地图、3D 拓扑 |
| 排行榜 | 列表/排行物料 + 标题 + 数字卡（总数） | 边框 |
| 详情页 | 表格 + 多图表 + 标题 | 标签、操作按钮 |

### 3.2 组合方案生成算法

```python
def generate_combinations(candidates, required_categories, max_materials=6):
    """
    1. 按分类（数字卡/图表/列表/...）从候选池选 Top 1
    2. 计算物料间兼容性（步骤 4）
    3. 如果不兼容，尝试候选池次选
    4. 物料数 > max_materials 截断
    5. 输出 Top 3 方案
    """
```

### 3.3 方案输出格式

```json
{
  "scenario": "销售大盘",
  "schemes": [
    {
      "rank": 1,
      "label": "🟢 标准方案",
      "materials": [
        { "name": "digital-flop", "role": "核心指标（销售总额）" },
        { "name": "echarts-bar", "role": "销售排行 Top10" },
        { "name": "echarts-pie", "role": "品类占比" },
        { "name": "normal-label", "role": "标题" },
        { "name": "decoration/border1", "role": "边框装饰" }
      ],
      "composition_cost": "低",
      "compatibility_score": 4.5,
      "estimated_total_minutes": 45,
      "needs_custom_dev": false
    }
  ]
}
```

## 步骤 4：组合代价计算

### 4.1 兼容性矩阵

物料两两间的兼容性是**对称的**，从画像的 `composability` 字段读取：

```json
{
  "compatible_with": [
    { "material": "digital-flop", "cost": "低", "reason": "同主题、数据流独立" }
  ]
}
```

### 4.2 兼容性判定维度

| 维度 | 说明 | 兼容性影响 |
|---|---|---|
| **数据格式一致** | 两个物料能否消费同一份 dataSource | 关键 |
| **主题/样式一致** | 颜色/字体/边框是否能统一配置 | 中 |
| **尺寸/布局一致** | 默认 width/height 是否能拼接 | 中 |
| **交互链通** | 一个的下钻事件能否被另一个消费 | 高级 |
| **同分类** | 同分类物料通常可替换 | 加分 |

### 4.3 组合代价等级

| 代价等级 | 含义 | 判定 |
|---|---|---|
| **🟢 低** | 直接组合，无需任何额外配置 | 兼容性矩阵全部"低" + 无需胶水代码 |
| **🟡 中** | 需要少量配置（如统一主题、调尺寸） | 至多 1-2 个"中"代价 + 无需胶水代码 |
| **🔴 高** | 需要写胶水代码 / 数据预处理 / 深度定制 | 任意"高"代价 或 需二次开发 |
| **⚫ 不可组合** | 数据格式完全冲突或语义冲突 | 硬性不兼容 |

### 4.4 兼容性评分公式

```
compatibility_score = 5 - 平均(所有物料对代价)
其中代价数值映射：低=0, 中=1.5, 高=3, 不可=5
```

## 步骤 5：方案评级

### 5.1 方案评级维度

| 维度 | 权重 | 评分依据 |
|---|---|---|
| **业务贴合度** | 30% | 方案是否真正讲清了输入的业务场景 |
| **物料数量合理性** | 20% | 物料数越少越好（≤5 满分，6-7 良好，8+ 一般）|
| **组合代价** | 25% | 全部"低" = 5 分；任一"高" = 2 分 |
| **搭建总成本** | 15% | 累计 `estimated_minutes` < 30 = 5 分；30-60 = 4 分；>60 = 2 分 |
| **是否需要二开** | 10% | 否 = 5 分；是 = 2 分 |

### 5.2 方案等级标签

| 总分 | 标签 | 含义 |
|---|---|---|
| 4.5+ | 🟢 **直接可搭** | 给 PM 选这套即可，无需研发额外投入 |
| 3.5-4.5 | 🟡 **需少量配置** | 大部分能搭，少量需要研发配合调 |
| 2.5-3.5 | 🔴 **需深度开发** | 物料齐全但需要不少胶水代码 |
| <2.5 | ⚫ **不建议组合** | 找替代物料或重新设计场景 |

### 5.3 方案输出示例

```json
{
  "scheme": {
    "rank": 1,
    "scenario": "销售大盘",
    "label": "🟢 直接可搭",
    "grade": "A",
    "score": 4.6,
    "materials": [
      { "name": "digital-flop", "role": "核心指标", "cost_minutes": 10 },
      { "name": "echarts-bar", "role": "销售排行", "cost_minutes": 15 },
      { "name": "echarts-pie", "role": "品类占比", "cost_minutes": 12 },
      { "name": "normal-label", "role": "标题", "cost_minutes": 3 },
      { "name": "decoration/border1", "role": "装饰", "cost_minutes": 2 }
    ],
```

## 特殊场景处理

### 单物料就能搞定

如果 `match_score` 最高的物料 `rating.label = "🟢 独立优秀"`，且**业务场景本身比较单一**（如"看实时销售总额"），**直接推荐单物料**：

```json
{
  "scheme": {
    "rank": 1,
    "scenario": "实时销售总额",
    "label": "🟢 单物料即可",
    "single_material": "digital-flop",
    "reasoning": "数字卡本身能独立讲清业务，无需组合",
    "total_minutes": 5
  }
}
```

### 无合适物料

如果候选池为空（或 `match_score` 全部 < 0.3），**给出"无合适方案"反馈 + 提示**：

```json
{
  "scheme": null,
  "scenario": "VR 设备拓扑",
  "feedback": {
    "level": "warning",
    "message": "未找到匹配物料",
    "suggestions": [
      "考虑使用 twaver-topo（3D 拓扑）或 oss-gis（地图）作为近似方案",
      "VR 拓扑可能需要二开，建议联系研发评估"
    ]
  }
}
```
