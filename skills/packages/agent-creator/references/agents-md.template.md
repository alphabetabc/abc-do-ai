# `agents.md` 模板

> 这份模板可以被：
>
> - 本 skill 在 Bootstrap 时直接套用
> - 其他 skill 在自己的 SKILL.md 里通过链接引用
> - 同事手动复制到新项目
>
> 复制后**逐项替换占位符**，不要保留 `<...>` 标记。

---

```md
# <项目名> — Agent Brief

<!--
这是项目给 agent 看的"门面文档"。
所有进入项目的 agent 都应先读这一份。
-->

- 业务一句话：<例如：电力应急指挥调度>
- 主要用户：<例如：省级电网调度员>
- 关键入口：<URL / 命令 / 文档链接>

## 约定

<!-- 不写在这里的话，agent 不知道，做出来就乱 -->

- 包管理器：<pnpm / uv / ...>
- 目录约定：<src/ vs packages/ 怎么分>
- 命名规则：<camelCase / kebab-case / ...>
- **不要做的事**：<例如：不要直接 commit 到 main>
- 常用命令：<例如：pnpm test / make dev>

## 领域语言

<!-- 项目特有的术语，agent 必须用这些词，不要用同义词；通用编程概念不写在这里 -->

| ------------ | ----------------------------------- |
```

---

## 填写提示

- **不要把 agents.md 写成 README 的副本**。README 给人看，agents.md 给 agent 看。
- **不要在这里写"我自己也记不住"的细节**——那些属于 `research/` 笔记。
- **不要把"做了就回不去"的决策埋在聊天记录里**——必须落 `decisions/NNNN-*.md`，并在这里登记。
- **领域语言只写项目特有术语**。通用编程概念（timeout、error handling 等）不属于这里。
- **`decisions/` 文件结构**：每个文件三段——`## 背景`（为什么需要决策）、`## 决策`（选了什么、不选什么、为什么）、`## 后果`（带来了什么限制或影响）。文件名 `NNNN-<title>.md`，NNNN 递增编号。
