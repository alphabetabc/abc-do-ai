# 通用约束句与反例

> 任何场景可叠加的"通用约束句"和"反例速查"。
>
> 用法：在反向提问、挂文档实现、任意会话中，按需要复制下面三句之一加到提示词末尾。
>
> 阶段提示词（反向提问、五件套生成、按文档实现、协助查现状等）见：
>
> - 新项目：[spec-five-pieces.md](./spec-five-pieces.md)
> - 历史项目：[legacy-change-template.md](./legacy-change-template.md)

## 通用约束三句（随时可加）

### 1. 收紧范围

```text
本次任务范围仅限：{一句话范围}。
明确不做：{非目标列表}。
不要重构无关模块；不要「顺便优化」未点名的文件。
有疑问先问，不要自行扩展范围。
```

### 2. 强迫文档同步

```text
若本次变更改变了对外 API、持久化或用户可见行为，列出需同步的文档清单
（spec / acceptance-tests / design / OpenAPI），并说明每项改了什么。
禁止：契约已变但仅更新代码。
```

### 3. 发现未知就停

```text
遇到文档未写清的行为或字段：先停下来提问，列出未知点，
不要用「业界最佳实践」或猜测补全后继续大改。
```

## 反例（不要这样写提示词）

| 烂提示词 | 问题 |
|----------|------|
| 「帮我把这块做漂亮一点，参考业界最佳实践，你看着改就行。」 | 无权威文档、无边界、默许臆造与大改 |
| 「顺便把相关模块也重构一下。」 | 扩大非目标；历史仓极易改炸 |
| 「按旧 wiki 实现就行。」 | 裸 P4；文档常与线上不符 |
| 「运维说一直这样，直接按他说的改。」 | 裸 P5；口头须核实 |
| 只贴聊天记录、不挂仓库文档路径 | AI 下次会话丢失权威；也无法评审 |

**好习惯一句：** 提示词里写上文档路径 + 非目标 + 契约以哪份 design/OpenAPI 为准。

## 阶段提示词（按需复制）

### 新项目 — 补验收自动化（验收写完后）

```text
请根据 {路径}/acceptance-tests.md 中的场景，在现有测试框架下补充自动化
（或可重复的手工验收步骤记录）。
优先覆盖关键路径与契约边界；不要为尚未写入验收文档的行为编写测试。
```

### 历史项目 — 协助查清现状（提示词全文见 legacy-change-template.md §提示词）

简版：

```text
我在为遗留变更写现状（as-is）。请按 P1–P6 取证办法帮我整理，不要编造未观察到的行为。
已知：【粘贴现象 / 链接 / 文件路径 / 日志摘要 / 旧文档片段】
请输出：
1) 主要办法与辅助办法（仅从 P1 自己跑 / P2 测试·注释等旁证 / P3 读代码 /
   P4 整理旧文档可 AI 归纳 / P5 问运维或产品 / P6 看线上证据 中选）；
2) 一份可写入模板的现状草稿（主流程 + 边角 + 本期先不修的坑）；
3) 仍未知、必须用人复现或问运维/产品的清单；
4) 禁止假设线上行为；不知道就标【未知】。
```
