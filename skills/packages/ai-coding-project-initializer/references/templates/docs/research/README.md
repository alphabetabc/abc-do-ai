# 调研备忘

## 与 design 的边界

- 这里是非正式基线：备选方案对比、PoC、抓帧、第三方文档摘录、竞品/协议探查
- **采纳为架构或 API 行为时必须回写** `docs/design/` 或 ADR
- 五件套只引用 design，不引用"只有 research 里写过"的隐性契约

## 可以放 / 不要放

| 可以放 | 不要放 |
|--------|--------|
| 多方案对比、实验结论、风险记录 | 已采纳的架构结论（应回写 design） |
| 第三方行为探查笔记 | 密钥、未脱敏生产数据 |
| 给写五件套用的参考材料 | 让 spec 直接依赖外部实现细节当契约 |
