## A. 功能名片

- 名称：用户资料管理
- 一句话价值：让用户自助维护姓名、头像、联系方式，减少管理员人工修改
- 入口：用户中心 → 资料编辑
- 优先级：P1
- 依赖：账号系统（已有）、对象存储（已有，用于头像）

## B. 背景与范围

- 痛点：当前用户资料只能找管理员改，工单占比 15%
- 本期目标：用户可自助编辑姓名、头像、手机号；管理员可查看修改记录
- 明确不做：密码修改（走账号中心）、角色权限变更（走管理后台）

## C. 用户与权限

- 业务角色与可见能力：
  - 普通用户：编辑自己的资料
  - 管理员：查看所有用户的资料 + 修改记录
- 隐藏 / 禁用 / 跟全站规范：遵循 Shell 的路由守卫与按钮权限

## D. 用户旅程

<!-- 5–10 步；关键分支单独写 -->

1. 用户登录后点击右上角头像 → 进入"个人资料"页
2. 页面展示当前姓名、头像、手机号（脱敏显示 138****1234）
3. 点击"编辑"→ 表单可编辑
4. 修改姓名 → 失焦校验长度（2-20 字符）
5. 上传头像 → 裁剪为正方形 → 上传对象存储
6. 修改手机号 → 发送验证码 → 校验
7. 点击"保存"→ 提交 → 成功提示
8. 失败分支：网络错误 → 保留编辑态，提示重试

## E. 界面清单

| 元素 | 说明 |
|------|------|
| 资料展示卡片 | 只读态：头像 + 姓名 + 脱敏手机号 |
| 编辑表单 | 姓名 input + 头像上传 + 手机号 input + 验证码 input |
| 保存按钮 | 表单校验通过后可点 |
| 修改记录列表（管理员） | 表格：时间 / 用户 / 字段 / 旧值 / 新值 |

## F. 数据与规则

<!-- 业务词；已知表字段可附；未知标「待确认」，禁止瞎编 -->

- 用户实体已有字段：`id`、`name`、`avatar_url`、`phone`、`updated_at`
- 新增字段：`phone_verified_at`（手机号验证时间）
- 规则：
  - 姓名：2-20 字符，不含特殊符号
  - 头像：≤ 2MB，jpg/png，裁剪为 200x200
  - 手机号：11 位数字 + 短信验证码校验
  - 修改记录保留 90 天

## G. 验收标准

<!-- S1… 可观察；可选 Gherkin；每条都能失败/通过 -->

- S1：普通用户可编辑自己的姓名、头像、手机号
- S2：手机号变更必须通过验证码校验
- S3：管理员可在"修改记录"页查看任意用户的资料变更历史
- S4：姓名超长或含特殊符号时，表单失焦校验失败并提示
- S5：头像超过 2MB 时，上传前拦截并提示
